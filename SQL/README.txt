Непосредственно для работы приложения спроектировано 6 сущностей:

1. Поезд - содержит два поля: id - номер поезда, являющееся первичным ключом (предполагается, что номер поезда не может быть изменен впоследствии, так же как и не может быть двух поездов с одинаковым номером) и places_qty - вместимость (число мест в поезде).

2. Станция - содержит три поля: id - ключевое автоикрементируемое поле, title - название станции, time_zone - часовой пояс станции (выраженный в текстовом представлении, например: "GMT+03:00").

3. Пассажир - содержит четыре поля - id - ключевое автоикрементируемое поле, first_name и last_name - соответственно, имя и фамилия, birthdate - дата рождения.

4. Таблица связей "Поезд - Станция" (train_station) - содержит список уникальных значений "Поезд - Станция" и порядковый номер станции на маршруте поезда. 1 - означает станцию отправления, 2,3 и т.д. - следующие станции. 

5. Расписание (board) - содержит 5 полей: id - ключевое автоикрементируемое поле, trip_number - номер рейса поезда, train_station_id - ссылка на информацию о поезде и станции, arrive_time - время прибытия поезда на станцию, departure_time - время отправления поезда со станции. Время записывается в формате Timestamp (в формате UTC), поправка на часовой пояс происходит при выводе расписания. 

6. Билет - содержит 4 поля: id - ключевое автоикрементируемое поле (можно сказать, номер билета), passenger_id - ссылка на пассажира, departure - ссылка на информацию об отправлении (строчка в расписании: номер поезда, рейс, станция, время) arrive - ссылка на информацию о прибытии (так же как и с отправлением).


Также БД включает в себя 2 вспомогательные сущности:

7. Пользователь (менеджер или админ) - содержит id, логин, пароль (в этом поле храниться хеш-сумма пароля), роль.

8. Таблица ролей - пока там будут только "менеджер" и "администратор", в дальнейшем может расшириться. Таблица включает в себя 2 поля: id и название роли. Опционально можно добавить сюда поле с правами, но это уже если время останется.
