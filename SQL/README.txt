Для приложения спроектировано 5 сущностей:

1. Поезд - содержит два поля: id - номер поезда, являющееся первичным ключом (предполагается, что номер поезда не может быть изменен впоследствии, так же как и не может быть двух поездов с одинаковым номером) и places_qty - вместимость (число мест в поезде).

2. Станция - содержит три поля: id - ключевое автоикрементируемое поле, title - название станции, time_zone - часовой пояс станции.

3. Пассажир - содержит четыре поля - id - ключевое автоикрементируемое поле, first_name и last_name - соответственно, имя и фамилия, birthdate - дата рождения.

4. Расписание (board) - содержит 5 полей: id - ключевое автоикрементируемое поле, train_id - номер поезда, station_id - id станции, arrive_time - время прибытия поезда на станцию, departure_time - время отправления поезда со станции. 

По своей сути эта таблица содержит в себе глобальное расписание для всех поездов и всех станций. Если нужно получить расписание всех поездов на определенной станции, надо сделать select по ней с сортировкой времени отправления поезда. Если нужно получить расписание и маршрут для определенного поезда - select по поезду с сортировкой времени отправления/прибытия.

Время отправления/прибытия содержатся в "абстрактном виде", отсчет - как в Unix-time - с 1 января 1970 года. К примеру, скорый поезд №35 Спб-Адлер с отправлением из Питера в 20:07 (дата в базе будет 1.01.1970), в Воронеже он будет в 12:58 следующего дня (дата 02.01.1970), в Адлер он прибудет еще через день в 10:14 (дата 03.01.1970). Таким образом, для получения времени и дат для конкретного поезда нужно будет имеющиеся в базе дату/время прибавить к 00:00:00 целевой даты + поправка на часовой пояс, если маршрут движения подразумевает выход за пределы одного ЧС.

5. Билет - содержит 5 полей: id - ключевое автоикрементируемое поле (можно сказать, номер билета), passenger_id - ссылка на пассажира, departure_station - станция отправления, arrive_station - пункт назначения, departure_date - дата отправления.

Я думаю, что проверку, чтобы в departure_station и arrive_station был одинаковый номер поезда, следует реализовать в логике приложения, а не в БД.
